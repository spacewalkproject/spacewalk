import com.sleepycat.db.*;

Description

This manual page describes the DbMemoryException class and how it is used by the various Db* classes.

A DbMemoryException is thrown when there is insufficient memory to complete an operation, and there is the possibility of recovering. An example is during a Db.get or Dbc.get operation with the Dbt flags set to Db.DB_DBT_USERMEM. In this case, the DbMemoryException.get_dbt method returns the problematic Dbt, and the operation can be retried with a larger buffer.
